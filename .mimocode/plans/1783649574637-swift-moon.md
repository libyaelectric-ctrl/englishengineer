# Plan: vocabulary.data.ts ‚Üí JSON Refactoring

## Goal
Move ~2950 lines of hardcoded vocabulary row data from `vocabulary.data.ts` into `vocabulary.data.json`, reducing the TS file from ~3700 lines to ~730 lines (types + row() + expansionCategories + processing + export).

## Current State
- `vocabulary.data.ts` (3691 lines): imports, types, `row()` helper, `BEGINNER_CONTENT_ROWS` (lines 46-181), `CONTENT_ROWS` (lines 183-3000), `expansionCategories` (lines 3002-3642), processing logic, export
- `tsconfig.json`: `resolveJsonModule` is NOT enabled ‚Äî needs enabling
- `vite.config.ts`: no special JSON config needed (Vite handles it natively)
- Sole export: `VOCABULARY_ENTRIES: VocabularyEntry[]`
- 5 direct consumers of the file (service + 3 test files + barrel index)

## Steps

### Step 1: Enable `resolveJsonModule` in `tsconfig.json`
- Add `"resolveJsonModule": true` to the compiler options
- Path: `tsconfig.json`

### Step 2: Generate `vocabulary.data.json`
- Read all `row(...)` calls from `BEGINNER_CONTENT_ROWS` and `CONTENT_ROWS`
- Convert each positional `row(word, pos, cefr, discipline, meaning, definition, example, synonyms, collocations, difficulty, tags)` into a named-key object
- Output a JSON array of objects matching the `VocabularyContentRow` shape
- Path: `src/features/vocabulary/vocabulary.data.json`

### Step 3: Update `vocabulary.data.ts`
- Remove the ~2950 lines of `BEGINNER_CONTENT_ROWS` and `CONTENT_ROWS` definitions (lines 46-3000)
- **KEEP** the `row()` helper function ‚Äî `buildExpansionRows()` at line 3647 still uses it
- Add: `import rawData from './vocabulary.data.json';`
- Split the imported JSON into beginner and non-beginner arrays based on `difficulty === 'Beginner'`
- Keep ALL other logic intact: `VocabularyContentRow` interface, `PartOfSpeech` type, `ExpansionCategory` type, `expansionCategories`, `buildExpansionRows()`, `dedupeRowsByWord()`, merge logic, `VOCABULARY_ENTRIES` export

**Key detail**: The original code has `BEGINNER_CONTENT_ROWS` as a separate array that participates in the merge logic (beginner rows override leveled rows). We need to preserve this by splitting the JSON data:
```ts
import rawData from './vocabulary.data.json';
const allRows = rawData as VocabularyContentRow[];
const BEGINNER_CONTENT_ROWS = allRows.filter(r => r.difficulty === 'Beginner');
const CONTENT_ROWS = allRows.filter(r => r.difficulty !== 'Beginner');
```

### Step 4: Verify
- Run `npx tsc --noEmit` to verify type safety
- Run `npm run test` if available
- Confirm `VOCABULARY_ENTRIES` export is unchanged
