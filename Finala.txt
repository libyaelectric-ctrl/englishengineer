## MADDE 1 — PricingPage Lint Hatası

Dosya: `src/pages/PricingPage.tsx:344`
Hata: `` `tabIndex` should only be declared on interactive elements
jsx-a11y/no-noninteractive-tabindex ``

Şu satırı bul:
```tsx
<div className="mt-4 overflow-x-auto rounded-[4px] border border-[#d9d9e3] shadow-sm" tabIndex={0} role="region" aria-label="Plan comparison table">
```

Şununla değiştir (rol'ü ESLint'in kabul ettiği bir düzene çevir, satırın
hemen üstüne gerekçeli disable yorumu ekle):
```tsx
{/* WCAG 2.1 SC 2.1.1: scrollable region needs keyboard focus for users who cannot use a mouse to scroll horizontally */}
{/* eslint-disable-next-line jsx-a11y/no-noninteractive-tabindex */}
<div className="mt-4 overflow-x-auto rounded-[4px] border border-[#d9d9e3] shadow-sm" tabIndex={0} role="region" aria-label="Plan comparison table">
```

**Doğrulama:** `npm run lint` çıktısında bu dosya için hata görünmemeli.

## MADDE 2 — cn.test.ts Lint Hatası

Dosya: `src/shared/utils/cn.test.ts:10`
Hata: `Unexpected constant truthiness on the left-hand side of a && expression`

Şu satırı bul:
```ts
expect(cn('foo', false && 'bar', 'baz')).toBe('foo baz');
```

Şununla değiştir:
```ts
const isActive = false;
expect(cn('foo', isActive && 'bar', 'baz')).toBe('foo baz');
```

**Doğrulama:** `npm run lint` → **toplamda 0 error** (uyarılar kalabilir).
`npm run test` → `cn.test.ts` hâlâ geçiyor.

## MADDE 3 — Repo Hijyeni: Kök Dizindeki Yabancı Dosyalar

Şu dosyalar public repoda durmamalı:
```
VC Checklist.docx
pricing_matrix.xlsx
gece.png
gecce.txt
gceci.txt
~$konsol.txt
```

**İşlem:**
1. Bu dosyaları `git rm` ile kaldır (sadece silme değil, git'ten kaldır):
   ```bash
   git rm "VC Checklist.docx" pricing_matrix.xlsx gece.png gecce.txt gceci.txt "~\$konsol.txt"
   ```
2. `.gitignore`'a şunu ekle ki bir daha yanlışlıkla commit edilmesinler:
   ```
   *.docx
   *.xlsx
   ~$*
   gece.png
   ```
   (Eğer proje meşru olarak .docx/.xlsx üretiyorsa — ör. rapor
   şablonları — bu satırları kaldır, sadece bu 6 dosyayı spesifik
   olarak ignore et.)
3. **ÖNEMLİ:** Bu dosyalar zaten public git geçmişinde duruyor,
   sadece son commit'ten silmek geçmiş commit'lerde hâlâ erişilebilir
   bırakır. Eğer bu dosyalarda gerçekten hassas iş/finans bilgisi
   varsa (VC Checklist, pricing_matrix içerikleri gerçek rakamlar
   içeriyorsa), `git filter-repo` veya BFG Repo-Cleaner ile **tüm git
   geçmişinden** kalıcı olarak temizlenmesi gerekir — bu, repo'nun
   force-push edilmesini gerektirir ve geri dönüşü olmayan bir işlemdir,
   bu adımı yapmadan önce kullanıcıya sor ve onay al.

**Doğrulama:**
- `git log --all --full-history -- "VC Checklist.docx"` çıktısını göster
  (geçmişte olduğunu kanıtla, karar kullanıcıya bırakılacak).
- `ls` (kök dizin) çıktısında bu 6 dosyanın artık görünmediğini göster.
