- `fail_count` (int): default 0
- `correct_count` (int): default 0
- `last_practiced_at` (timestamptz)
- `mastered_at` (timestamptz)

Migration dosyası: `supabase/migrations/add_vocabulary_progress_fields.sql`

---

**GÖREV 2: DURUM GEÇİŞ MANTIĞI (OTOMATİK)**

**2.1 New → Learning (Otomatik):**
Kullanıcı kelime kartını görüntülediğinde (tıklama, hover, sayfada görünme) status otomatik 'learning' olsun. Ekstra tıklama yok.

**2.2 Learning → Learned (Doğru Cevap):**
Quiz/writing/reading'de doğru cevap → `correct_count + 1`, `correct_count >= 1` ise status = 'learned'.

**2.3 Learned → Mastered (3 Doğru + 7 Gün Hatasız):**
`correct_count >= 3` VE son 7 gün içinde `fail_count = 0` ise status = 'mastered', `mastered_at = NOW()`.

**2.4 Herhangi bir durumda yanlış cevap:**
`fail_count + 1`. Eğer `fail_count >= 5` ise status = 'struggling'. Eğer status 'learned' veya 'mastered' iken yanlış gelirse, status = 'learning' (gerileme).

**2.5 Struggling Havuzu:**
`status = 'struggling'` olan kelimeler ayrı sekmede gösterilsin. Kullanıcı doğru cevapladıkça `fail_count` sıfırlanabilir, status tekrar 'learning' olabilir.

**GÖREV 6: BACKEND & FRONTEND ENTEGRASYONU**

**Backend (API):**
- Güncelle: `POST /api/vocabulary/:id/progress`
  - Body: `{ result: 'correct' | 'incorrect' }` veya `{ status: 'learning' | 'learned' | 'mastered' | 'struggling' }`
  - Backend yukarıdaki kurallara göre `fail_count`, `correct_count`, `status`, `mastered_at`, `last_practiced_at` günceller.
- Yeni: `GET /api/vocabulary/stats` → kullanıcının istatistikleri (toplam, learning, learned, mastered, struggling).

**Frontend (Zustand Store):**
- `vocabulary.store.ts`'ye ekle:
  - `updateWordProgress(wordId, result)` → backend isteği + store güncellemesi
  - `fetchVocabularyStats()` → istatistikleri çek
- `VocabularyPage.tsx`'de sağ yan bar bileşenini güncelle.
