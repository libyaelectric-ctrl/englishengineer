# EngineerOS v3.0 UI / UX Bible

## Product Feeling

EngineerOS should feel like a premium engineering operating system: calm, precise, focused and trustworthy.

It should not feel like a generic LMS, a playful language app or a developer dashboard.

## Visual Principles

- Default theme: light workspace
- Dark mode: optional for deep engine workspaces
- Spacing: generous, structured and consistent
- Shadows: soft and functional
- Gradients: minimal
- Corners: engineered, not toy-like
- Buttons: 12px radius
- Cards: 16px radius
- Inputs: 12px radius
- Dialogs: 20px radius

## Information Architecture

Every screen should answer:

- Where am I?
- What should I do now?
- How much progress have I made?
- What unlocks next?
- How close am I to my professional goal?

## Navigation Standard

Primary navigation is limited to 8 entries:

- Home
- AI Copilot
- Reading
- Writing
- Listening
- Speaking
- Vocabulary
- Profile

Secondary routes remain available through contextual UI and direct routes.

## Home Standard

Home must prioritize:

- Today’s Mission
- Continue Learning
- Today’s Progress
- AI Recommendation
- Career Goal
- Mission Control

## Mission Control Standard

Mission Control may show no more than 10 visible items:

- Current CEFR
- Engineer Level
- XP
- Mission Streak
- Today’s Goal
- Weakest Skill
- Strongest Skill
- AI Recommendation
- System Status
- Billing Status

## Trust Philosophy

Never hide reality.

Clearly label:

- Local Mode
- Demo Mode
- Backend unavailable
- Billing local fallback
- Production mode when configured
- Sync requirements

## Interaction Philosophy

Interactions should be subtle:

- Smooth hover
- Clear focus
- Small elevation changes
- No distracting animation
- No sound by default

## Commercial UX Rules

- One primary action per screen.
- Empty states must explain value, not apologize.
- Upgrade messaging must be clear and non-aggressive.
- Do not expose future content too early.
- Do not use developer terminology in user-facing product surfaces.
