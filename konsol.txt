# Sayfa Konsolidasyon Planı (Mimo için)

**Hazırlanma tarihi:** 15 Temmuz 2026
**Konu:** `src/pages/` altında birbirine benzeyen/çakışan işi yapan sayfaların birleştirilmesi
**Kim için:** Bu dosya bir AI kod ajanının (Mimo) veya bir geliştiricinin tek başına, adım adım takip ederek uygulayabileceği şekilde yazıldı. Her adımda **ne yapılacağı**, **neden yapıldığı**, **hangi dosyaya dokunulacağı** ve **nasıl doğrulanacağı** ayrı ayrı belirtildi.

> ⚠️ **En önemli kural:** Bu plandaki hiçbir adım "tahmin" ile yapılmamalı. Her adımdan önce ilgili dosya gerçekten açılıp okunmalı (satır numaraları/içerik zamanla değişmiş olabilir). Her adımdan sonra `npm run test`, `npm run typecheck`, `npm run lint` çalıştırılıp **hepsi yeşil olmadan bir sonraki adıma geçilmemeli**. Emin olunmayan bir nokta varsa, tahmin yürütüp devam etmek yerine durup soru sorulmalı.

---

## 📋 Önce: Neden bu değişiklik yapılıyor?

Kod tabanında 5 ayrı sayfa (`DashboardPage`, `ProgressPage`, `AnalyticsPage`, `GamificationPage`, `LearningIntelligencePage`) kullanıcıya aynı temel şeyi farklı şekillerde gösteriyor: **"Ne kadar ilerledim, seviyem ne, rozetlerim/hedeflerim neler?"**

Bunun kanıtı:
- Hepsi aynı state kaynağını (`useLearningStore` from `@/core/learning`) okuyor.
- 3 tanesi (`Analytics`, `Gamification`, `LearningIntelligence`) ana navigasyon menüsünde **hiç yok** — kullanıcı bunlara ancak başka sayfaların içine gömülü linklerden tesadüfen ulaşabiliyor.
- `ProgressPage.tsx`, `CurriculumPage`'in kendi iç veri dosyasından doğrudan import yapıyor (`./CurriculumPage/curriculum-data`) — bu, iki sayfanın aslında birbirinden kopyalanmış/birbirine bağımlı olduğunun kanıtı.

Ayrıca `AIPage` bileşeni hem `/ai` bağımsız sayfası olarak hem de `/tools/ai` sekmesi olarak **iki kere** tanımlanmış; aynı iş için iki farklı giriş noktası var.

**Hedef:** Kullanıcı deneyimini bozmadan (aynı özellikler kaybolmadan), bu dağınık yapıyı `ToolsPage`'de zaten kanıtlanmış olan "tek sayfa + sekmeler" pattern'iyle birleştirmek.

---

> **Güncelleme (2. tur karar):** İçerik başlıkları tek tek incelendi ve gerçek çakışmalar bulundu — örneğin "Achievement Feed" (Gamification) ile "Recent Achievements" (Analytics) aynı veriyi iki kez gösteriyordu; "Weak Skills" (Analytics) ile Dashboard'daki "Needs attention" aynı şeydi. Bu yüzden plan **4 sekme yerine 2 sekmeye** indirildi. Aşağıdaki tüm adımlar bu 2-sekmeli yapıya göre güncellendi.

## 🔧 ADIM 1 — AI Copilot ikiliğini çöz (küçük, düşük riskli, önce bunu yap)

Bu en basit adım, önce bunu bitirip test etmek güven verir.

1. **Dosya:** `src/routes/router.tsx`
   - Şu bloğu bul:
     ```tsx
     const AI = lazy(() => import('@/pages/AIPage'));
     ```
     ve route tanımındaki:
     ```tsx
     {
       path: 'ai',
       element: (
         <Suspense fallback={<LoadingState />}>
           <AI />
         </Suspense>
       ),
     },
     ```
   - Bu route bloğunu **tamamen sil**. `const AI = lazy(...)` satırını da sil (artık kullanılmıyor).
   - Geriye kalan olası eski linkler kırılmasın diye, silinen route'un yerine bir yönlendirme koy:
     ```tsx
     {
       path: 'ai',
       element: <Navigate to="/tools/ai" replace />,
     },
     ```

2. **Dosya:** `src/shared/utils/prefetch.ts` ve nav ile ilgili diğer dosyalarda `/ai` (tam eşleşme, `/tools/ai` değil) geçen başka referans var mı kontrol et:
   ```bash
   grep -rn "'/ai'\|\"/ai\"" src --include="*.ts" --include="*.tsx"
   ```
   Bulunan her referansı `/tools/ai` ile değiştir (yönlendirme yeterli olmayan, doğrudan link üreten yerler varsa).

3. **Doğrulama:**
   - `npm run typecheck`, `npm run lint`, `npm run test` yeşil.
   - Uygulamayı çalıştırıp `/ai` adresine gidildiğinde otomatik olarak `/tools/ai`'ye yönlendirildiğini manuel kontrol et (veya bir e2e test ekle: `tests/e2e/` altında benzer bir test varsa örnek alınabilir).

## 🔧 ADIM 3 — "Progress" hub sayfasını oluştur (2 sekme, asıl büyük iş, dikkatli ilerle)

**Hedef yapı — sadece 2 sekme:**

### 3.4 — Container'ı yaz

`src/pages/ProgressPage/index.tsx` dosyasını `ToolsPage.tsx`'teki mantığın aynısıyla, ama **2 sekmeli** olarak yaz:

```tsx
import { lazy, Suspense } from 'react';
import { useParams } from 'react-router-dom';
import { PageHeader } from '@/shared/components/PageHeader';
import { ProgressOverviewTab } from './ProgressOverviewTab';

const ProgressNextStepsTab = lazy(() =>
  import('./ProgressNextStepsTab').then((m) => ({ default: m.ProgressNextStepsTab }))
);

type ProgressSection = 'overview' | 'next-steps';

const ProgressPage = () => {
  const { section } = useParams<{ section: string }>();
  const activeTab: ProgressSection = section === 'next-steps' ? 'next-steps' : 'overview';

  return (
    <div className="space-y-7 animate-in fade-in duration-300 pt-12 sm:pt-0">
      <PageHeader
        title="Progress"
        description="İlerlemen ve şimdi ne yapman gerektiği — tek yerde."
        badgeText={activeTab === 'overview' ? 'OVERVIEW' : 'NEXT STEPS'}
        badgeColor="border-cyan-200 bg-cyan-50 text-cyan-700"
      />

      {activeTab === 'overview' && <ProgressOverviewTab />}
      {activeTab === 'next-steps' && (
        <Suspense fallback={<div className="h-64 animate-pulse rounded-xl border border-border-soft bg-surface-hover" />}>
          <ProgressNextStepsTab />
        </Suspense>
      )}
    </div>
  );
};

export default ProgressPage;
```

> ⚠️ Bu kod bir **taslak/örnek**tir — mevcut `ToolsPage.tsx` ile birebir tutarlı olacak şekilde, gerçek dosya açılıp stil/import kuralları (örn. `embedded` prop pattern'i) oraya göre uyarlanmalı. Kör kopyala-yapıştır yapma.

### 3.5 — Router'ı güncelle

**Dosya:** `src/routes/router.tsx`

Şu satırları:
```tsx
const Progress = lazy(() => import('@/pages/ProgressPage'));
const Analytics = lazy(() => import('@/pages/AnalyticsPage'));
const Gamification = lazy(() => import('@/pages/GamificationPage'));
const LearningIntelligence = lazy(() => import('@/pages/LearningIntelligencePage'));
```
şu şekilde değiştir:
```tsx
const Progress = lazy(() => import('@/pages/ProgressPage'));
```
(Diğer 3 satır tamamen silinsin, artık bu sayfalar ayrı dosya olarak yok.)

Route bloklarını (`path: 'progress'`, `path: 'analytics'`, `path: 'gamification'`, `path: 'learning-plan'`) şu şekilde değiştir:

```tsx
{
  path: 'progress',
  element: <Navigate to="/progress/overview" replace />,
},
{
  path: 'progress/:section',
  element: (
    <Suspense fallback={<LoadingState />}>
      <Progress />
    </Suspense>
  ),
},
// Eski URL'leri kıranlarla karşılaşmamak için yönlendirmeler:
{ path: 'analytics', element: <Navigate to="/progress/overview" replace /> },
{ path: 'gamification', element: <Navigate to="/progress/next-steps" replace /> },
{ path: 'learning-plan', element: <Navigate to="/progress/next-steps" replace /> },
```

> Eski route'ları (`/analytics`, `/gamification`, `/learning-plan`) **direkt silme** — bunlara yönlendirme koy. Çünkü kod içinde başka yerlerde (`CurriculumPage.tsx`, `LearningMemorySummary.tsx`, `ProfilePage/SkillsProgressSection.tsx`) bu eski path'lere `to="..."` şeklinde linkler var; bunları da güncellemen gerekiyor (aşağıya bak) ama yönlendirme bir güvenlik ağı olarak kalsın.

### 3.6 — İçe gömülü (deep-link) referansları güncelle
